JavaScript 六角課程​

JavaScript 是怎如何運行的​

程式語言分成兩種，直譯式和編譯式。 編譯式語言會在開發的過程中，就會在編譯先除錯，這類語言通常效能較佳。 JavaScript 是直譯式語言，代碼生成後就可以直接在環境上運行，在環境上才會進行除錯，錯誤會直接反應在環境中，像我們平常看到的 console error，這類語言通常不會強制定義型別。 JavaScript 直譯器轉換，會先語法單元話（Tokenizing）(圖一），將 JS 裡面的詞彙解析出來，透過抽象結構樹（AST）（圖二）來生成代碼。

執行的錯誤環境 LHS、RHS​

RHS (Right-hand side) 是取值來自於右側的變數上 LHS (Left-hand side) 賦予值到左側的變數上

RHS 錯誤是無法在編譯的時候噴錯 M，要到環境上才會發現。 LHS 錯誤是在編譯的時候噴的錯。

語法作用域（Lexical scope）​

function callName() {
  var Ming = '小明';
}
callName();
console.log(Ming)

會發生以下錯誤

靜態作用域：變數的作用與是在語法解析時，就已經確定作用域，且不會再改變。

動態作用域：變數的作用與在函式調用時才決定。

下面再舉個另外的例子：

var value = 1;
function f1() {
  console.log(value) // 語法作用域 答案是1，如果是動態作用域，答案會是2
}
function fn2() {
  var value = 2;
  fn1();
}
fn2()

執行環境與執行堆疊（Execution Context）​

Execution context 和被宣告的位置沒有關係，只和哪時候被呼叫有關。

範圍鏈 (Scope chain)​

當函式的本身沒有這個變數，就會向外尋找，但與執行環境沒有任何關係，只和這個函式在哪被宣告有關係，因為 JS 是語法作用域。
